# 🔒 Security Policy

## 📌 Scope / 适用范围

**中文：**  
本仓库为公开展示版本，仅用于展示系统架构设计与工程实现思路，不承载真实生产数据或业务逻辑。

**English:**  
This repository is a **public showcase version**, intended solely for demonstrating system architecture and engineering practices. It does **not** contain real production data or full business logic.

---

## 🗂 Data & Credentials / 数据与凭据说明

**中文：**
- 仓库中的账号、邮箱、手机号、令牌及业务数据均为示例数据  
- 所有示例数据仅用于演示流程，不具备真实访问权限  
- 请勿将这些信息用于任何真实系统或安全测试  

**English:**
- All accounts, emails, phone numbers, tokens, and business data in this repository are **mock/demo data**
- These values are used only for demonstration purposes and have **no real access privileges**
- Do **not** use them for any real system or security testing

---

## 🚨 Vulnerability Reporting / 漏洞反馈

如果你发现潜在的安全问题，欢迎通过 Issue 提交：

If you discover any potential security vulnerabilities, please report them via GitHub Issues with the following details:

- 影响范围 / Scope of impact  
- 复现步骤 / Steps to reproduce  
- 风险评估 / Risk assessment  
- 修复建议 / Suggested fix  

---

## ⏱ Response Policy / 响应说明

**中文：**  
我会在确认问题后尽快进行处理，并在必要时更新相关代码或说明。

**English:**  
After verification, I will address the issue as soon as possible and update the code or documentation if necessary.

---

## ⚠️ Disclaimer / 免责声明

**中文：**  
本项目不适用于生产环境。如需用于实际业务，请自行完善安全策略，包括但不限于鉴权、权限控制、数据加密、审计与监控等。

**English:**  
This project is **not production-ready**. If you intend to use it in a real-world scenario, you must implement proper security measures, including but not limited to authentication, authorization, encryption, auditing, and monitoring.

---
